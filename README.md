1. 本体管理器,需要增删改查
2. 本体属性，包括本体名，状态，权限信息
3. 本体会设置多个对象字段，每个字段属性包括字段名，类型，校验规则。id，name(name和id被标记成键值和标题)是默认必须的，其他属性可以通过页面自己添加保存到此对象中。
3.1. 本体的每个对应对象字段，都需要设置该字段的校验规则，可以自定义规则
4. 本体可以注册多个行动，这些actions是维护好的工具库，比如维护了ai预测的接口，这个ai预测的接口需要smiles）（注册时提供），token，和predict_type（工具库提供）。那么本体注册的时候需要将本体定义的属性与工具接口设置的字段做绑定。
5. actions工具库管理，可以设置不同的工具连接，用于在创建本体的时候绑定维护的工具库。
6. 数据源管理，可以配置需要同步的第三方数据源数据接口，增删改查，维护字段包括url，token/其他认证方式，请求参数配置，同步定时器，数据同步规则等等
7. 数据源与本体模型映射页面，选择要关联的本体，然后选择数据源后，可以调用接口将返回的数据结构快速显示，手动连接本体与数据源的关系。
8. 数据同步逻辑，前置体条件，创建本体，以及本体与本体的关联，数据源与本体的映射后，在定时器同步数据的时候，需要将数据源按照本体的结构存储到graphDB中，并关联其他的数据本体，同步的时候需要校验数据的规范，规则，如果设置了对应的actions，需要异步执行
9. 同步记录管理，记录数据同步的log，同步失败，校验失败的需要标记


注意
本体与本体可以建立关联，作为graph数据的node基础
