你是一个 高级前端架构师，需要为一个 自动化 Agent 测试平台设计前端页面。

一、系统背景

后端是一个 自动化测试编排系统，通过 HTTP API 测试 Agent 行为，并通过 WebSocket 推送实时进度。


二、前端核心需求
1️⃣ 测试运行监控（实时）

显示测试 Run 列表

每个 Run：

状态（pending / running / done / failed）

总进度条

当前并发用户数

2️⃣ 测试流程可视化（重点）

展示 Scenario 流程（步骤 / 节点）

每个虚拟用户一条“泳道”

节点状态：

running

success

failed

节点耗时可视化（颜色 / tooltip）

3️⃣ 实时进度推送

通过 WebSocket 接收事件

动态更新 UI

不刷新页面

4️⃣ 测试结果展示

Summary：

成功率

平均响应时间

失败节点分布

Detail：

单个用户执行轨迹

单个节点请求 & 返回内容

三、页面结构建议（必须输出）

请设计以下页面：

测试列表页

测试详情页（实时）

测试结果页（历史）

单用户执行轨迹页

四、技术建议

Vue3 任选

WebSocket 客户端

DAG / 流程图可用现有库

强调“工程可落地”

五、输出要求

给出页面结构说明

给出组件拆分建议

给出 WebSocket 数据格式示例
